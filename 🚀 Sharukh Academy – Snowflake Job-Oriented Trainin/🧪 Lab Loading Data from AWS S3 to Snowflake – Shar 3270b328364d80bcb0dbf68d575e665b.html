🔐 Step 1: Create Storage Integration

CREATEOR REPLACE STORAGE INTEGRATION s3_int
TYPE= EXTERNAL_STAGE
STORAGE_PROVIDER= S3
ENABLED=TRUE
STORAGE_AWS_ROLE_ARN='your-aws-role-arn'
STORAGE_ALLOWED_LOCATIONS= ('s3://bucket-name/path/')
COMMENT='Integration with AWS S3';

🧠 Key Concepts

  • ARN → Amazon Resource Name
  • IAM Role → Used for secure access
  • Avoids hardcoding credentials

🔍 Verify Integration

DESC INTEGRATION s3_int;

👉 Use output to configure AWS IAM role

🗄️ Step 2: Create Database & Schema

CREATE DATABASEIFNOTEXISTS MYDB;

CREATESCHEMAIFNOTEXISTS MYDB.file_formats;
CREATESCHEMAIFNOTEXISTS MYDB.external_stages;

📄 Step 3: Create File Format

CREATEOR REPLACE FILE FORMAT mydb.file_formats.csv_fileformat
TYPE= CSV
FIELD_DELIMITER='|'
SKIP_HEADER=1
EMPTY_FIELD_AS_NULL=TRUE;

📂 Step 4: Create External Stage

CREATEOR REPLACE STAGE mydb.external_stages.aws_s3_csv
URL='s3://your-bucket/path/'
STORAGE_INTEGRATION= s3_int
FILE_FORMAT= mydb.file_formats.csv_fileformat;

📋 Step 5: List Files

LIST @mydb.external_stages.aws_s3_csv;

👉 Confirms connection with S3

🧾 Step 6: Create Target Table

CREATEOR REPLACETABLE mydb.public.customer_data (
customerid NUMBER,
custname STRING,
email STRING,
city STRING,
state STRING,
DOBDATE
);

📥 Step 7: Load Data (COPY Command)

COPYINTO mydb.public.customer_data
FROM @mydb.external_stages.aws_s3_csv
PATTERN='.*customer.*';

✅ Step 8: Validate Data

SELECT*FROM mydb.public.customer_data;

🧠 End-to-End Flow

👉 S3 → Storage Integration → Stage → COPY → Table

⚠️ Common Errors

  • ❌ Wrong IAM Role / ARN
  • ❌ Incorrect S3 path
  • ❌ File format mismatch
  • ❌ Permission issues

🎯 Best Practices

  • Use Storage Integration (secure)
  • Organize files properly in S3
  • Use pattern to filter files
  • Validate after loading
