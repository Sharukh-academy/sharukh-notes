🧊 What is a Virtual Warehouse?

A Virtual Warehouse (VW) is a cluster of compute resources used to:

  • Run SQL queries
  • Perform DML operations
  • Load data

👉 It is the compute engine of Snowflake

⚙️ Types of Workloads

Different warehouses can be used for different purposes:

  • 📥 Data Loading Warehouse
  • 📊 Query Processing Warehouse

👉 Best practice: Separate warehouses for separate workloads

🧠 How Virtual Warehouses Work

  • Each warehouse consists of multiple compute nodes
  • On AWS → EC2 instances
  • Warehouse size determines:
    • Number of servers
    • Performance
    • Cost

📏 Warehouse Sizes

Warehouses come in multiple sizes:

  • XS → Small → Medium → Large → XL

Key Concept:

  • Each size increase =

    👉 Double the compute power

    👉 Double the cost

⏳ Query Execution Behavior

  • If resources are insufficient:
    • Queries get queued
    • Execute when resources are available

👉 This affects performance during heavy usage

🚀 Scaling Options (Very Important)

Snowflake provides 2 types of scaling

🔼 1. Scale Up (Vertical Scaling)

Increase warehouse size

When to use:

  • Queries are slow
  • Large data processing

Example:

  • Small → Medium → Large

👉 More power, faster execution

🔁 2. Scale Out (Horizontal Scaling)

Increase number of clusters

When to use:

  • Many users running queries simultaneously
  • High concurrency

Feature:

  • Multi-cluster warehouse
  • Available in Enterprise Edition

👉 Prevents query queuing

📊 Auto Scaling Behavior

  • During peak hours:
    • Additional clusters are started automatically
  • During off-peak:
    • Extra clusters are stopped

👉 This optimizes performance + cost

💰 Cost Optimization Features

⏸️ Auto Suspend

  • Stops warehouse when idle

▶️ Auto Resume

  • Starts automatically when query runs

👉 Saves cost significantly

🎯 Best Practices

  • Use small warehouse for learning
  • Separate:
    • ETL workloads
    • Reporting queries
  • Enable auto suspend/resume
  • Monitor query performance before scaling

⚠️ Common Mistakes

  • Using large warehouse unnecessarily
  • Not enabling auto suspend → cost waste
  • Using single warehouse for all workloads
  • Ignoring query optimization
