🧩 What is a DBT Model?

A model in DBT is a .sql file that contains a SELECT query used to transform raw data into structured, analytics‑ready tables or views 1.

Key Highlights:

💡 Models define business logic using SQL.

⚙️ DBT compiles and executes them in your data warehouse (e.g., Snowflake).

📂 Models are organized into folders like staging and marts.

🔗 Dependencies are managed automatically using ref() and source() functions.

🚀 Types of Model Materializations

Materialization defines how DBT builds your model in the warehouse 1.

1️⃣ View – Logical layer, always up‑to‑date, lightweight.

2️⃣ Table – Physical table, faster for reporting, rebuilt each run.

3️⃣ Incremental – Updates only new or changed data.

4️⃣ Ephemeral – Temporary model used only during compilation.

Example:

{{ config(materialized='view') }}
select * from {{ source('raw','customers') }}

📁 Folder Structure & Defaults

DBT uses folder‑level configuration to set default materializations 1:

  • staging/ → creates views
  • marts/ → creates tables

Defined in dbt_project.yml:

models:
  project_name:
    staging:
      +materialized: view
    marts:
      +materialized: table

🔗 Model Dependencies & Lineage

DBT automatically tracks dependencies using ref() and source() 1:

RAW TABLES
   ↓
STAGING MODELS (Views)
   ↓
FACT MODELS (Tables)

Example:

select
    c.customer_id,
    count(o.order_id) as total_orders
from {{ ref('stg_customers') }} c
left join {{ ref('stg_orders') }} o
    on c.customer_id = o.customer_id
group by 1

🧮 DBT Seeds Overview

Seeds are CSV files stored in the data/ folder that load static reference data into your warehouse 1.

Key Points:

📊 Used for lookup or reference tables (e.g., order status).

⚙️ Run using dbt seed.

🧱 Can assign data types and target schema explicitly.

🔁 Use --full-refresh to reload updated seed data.

Example command:

dbt seed --full-refresh

🎯 Best Practices

✅ Save models in correct folders (staging/marts).

✅ Use ref() instead of hard‑coding table names.

✅ Always test transformations before running full refresh.

✅ Keep seed files version‑controlled.
