🧊 Overview

Data loading is the process of bringing data into Snowflake tables for analysis.

👉 Snowflake provides multiple efficient loading methods

📌 Types of Data Loading

Snowflake supports 2 main types:

1️⃣ Bulk Loading

2️⃣ Continuous Loading

📦 1. Bulk Loading

Used for loading large volumes of data at once

🔧 Method:

  • Uses COPY Command
  • Data comes from:
    • External Stage (cloud storage)
    • Internal Stage (Snowflake storage)

⚙️ Key Features:

  • Batch processing
  • Uses Virtual Warehouse (compute required)
  • Suitable for:
    • Historical data loads
    • Large datasets

📁 Data Sources:

  • AWS S3
  • Azure Blob Storage
  • Google Cloud Storage
  • Local files → Internal Stage

🔄 2. Continuous Loading (Snowpipe)

Used for real-time / near real-time data loading

⚡ Key Features:

  • Loads data automatically
  • Works with small data batches (micro-batches)
  • Data available within minutes

🧠 Important Points:

  • Serverless (no warehouse required)
  • Separate cost (based on usage)
  • Ideal for:
    • Streaming data
    • Live dashboards

🛠️ COPY Command (Core Concept)

Used to load data into Snowflake tables

📌 Syntax:

COPYINTO table_name
FROM @stage
file_format= (type='CSV')
files= ('file1.csv')
-- OR
pattern='.*filepattern.*';

⚙️ Key Options:

  • file_format → defines data format
  • files → specific files
  • pattern → load multiple files using pattern

🔄 Transforming Data During Load

Snowflake allows basic transformations while loading

Supported Transformations:

  • Column reordering
  • Column skipping
  • String functions
  • Expressions
  • Sequence / auto-increment fields

👉 Saves time → no separate transformation step needed

🔗 Other Data Loading Methods

Using external tools:

  • Matillion
  • Informatica
  • DataStage
  • Hevo
  • Azure Data Factory
  • Azure Synapse

👉 Used in real-world enterprise environments

🎯 Best Practices

  • Use Bulk Load for large datasets
  • Use Snowpipe for real-time data
  • Choose correct warehouse size
  • Validate data before loading
  • Use proper file formats

⚠️ Common Mistakes

  • Using Snowpipe for large bulk data
  • Wrong file format configuration
  • Not monitoring load errors
  • Using oversized warehouse (cost issue)
